## О задаче

**Предметная область: банковский маркетинг.**
Банк хочет лучше понимать своих клиентов и предлагать каждому именно те продукты (кредиты, карты, вклады и т.д.), которые ему нужны. В базе банка есть 41 различный продукт. Задача — научиться предсказывать, какие продукты клиент, скорее всего, захочет открыть.

**Гипотеза исследования:**
Поведение клиента (как он тратит деньги, какие операции совершает, сколько ему лет и т.д.) связано с тем, какими продуктами он пользуется. Если это правда, то по данным о клиенте можно предсказать его «портфель» продуктов.

**Формальная постановка:**
Multi-label классификация. 1 миллион клиентов, у каждого — набор числовых и категориальных признаков. Для 750 тысяч клиентов известно, какими продуктами из 41 они владеют. Модель должна для каждого клиента выдать 41 вероятность — от 0 до 1.

**Критерий качества:**
Бизнесу важно не просто угадать наличие продукта, а правильно ранжировать продукты по вероятности — на первом месте должен стоять тот продукт, который клиент действительно откроет

> **Метрика: Macro Averaged ROC-AUC**

## Структура репозитория

| Файл | Описание |
|---|---|
| `01_EDA.ipynb` | Разведочный анализ: пропуски, распределения, whale-эффекты, корреляции таргетов |
| `images` | Папка с графиками по разведочному анализу |
| `02_feature_engineering.ipynb` | Подготовка трёх датасетов: tree_nan, tree_zero, linear |
| `03_models.ipynb` | Подбор гиперпараметров (Optuna), финальное обучение, сабмит |

## Датасеты

| Датасет | Описание | Модели |
|---|---|---|
| **tree_nan** | `is_whale_` + `count_not_null` | LightGBM, CatBoost |
| **tree_zero** | NaN → 0 + `_isnan` флаги + `is_whale_` + `count_not_null`| XGBoost |
| **linear** | скейлинг + энкодинг + `_isnan` флаги + попарные-взаимодействия + `is_whale_` + `count_not_null` | Logistic Regression |

## Модели

LightGBM · CatBoost · XGBoost · Logistic Regression
